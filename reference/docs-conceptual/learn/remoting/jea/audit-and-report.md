## <a name="find-available-role-capabilities-on-the-machine"></a><span data-ttu-id="38159-112">컴퓨터에서 사용 가능한 역할 기능 찾기</span><span class="sxs-lookup"><span data-stu-id="38159-112">Find available role capabilities on the machine</span></span>

<span data-ttu-id="38159-113">JEA는 PowerShell 모듈 내의 `.psrc`RoleCapabilities**폴더에 저장된** 파일에서 역할 기능을 가져옵니다.</span><span class="sxs-lookup"><span data-stu-id="38159-113">JEA gets role capabilities from the `.psrc` files stored in the **RoleCapabilities** folder inside a PowerShell module.</span></span> <span data-ttu-id="38159-114">다음 함수는 컴퓨터에서 사용할 수 있는 모든 역할 기능을 찾습니다.</span><span class="sxs-lookup"><span data-stu-id="38159-114">The following function finds all role capabilities available on a computer.</span></span>

```powershell
function Find-LocalRoleCapability {
    $results = @()

    # Find modules with a "RoleCapabilities" subfolder and add any PSRC files to the result set
    Get-Module -ListAvailable | ForEach-Object {
        $psrcpath = Join-Path -Path $_.ModuleBase -ChildPath 'RoleCapabilities'
        if (Test-Path $psrcpath) {
            $results += Get-ChildItem -Path $psrcpath -Filter *.psrc
        }
    }

    # Format the results nicely to make it easier to read
    $results | Select-Object @{ Name = 'Name'; Expression = { $_.Name.TrimEnd('.psrc') }}, @{
        Name = 'Path'; Expression = { $_.FullName }
    } | Sort-Object Name
}
```

> [!NOTE]
> <span data-ttu-id="38159-115">여러 역할 기능에서 같은 이름을 공유하는 경우 이 함수의 결과 순서가 반드시 역할 기능이 선택된 순서인 것은 아닙니다.</span><span class="sxs-lookup"><span data-stu-id="38159-115">The order of results from this function is not necessarily the order in which the role capabilities will be selected if multiple role capabilities share the same name.</span></span>
